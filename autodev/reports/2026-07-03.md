## What was built

- `extension/dashboard.js` / `.html` / `.css` — a "✨ Tailor" button on every tracked-application
  row, opening `tailor.html` with the job's company/title as context.
- `extension/popup.html` / `.js` / `.css` — added a "⚙ Settings" link next to "Open dashboard".
- `extension/manifest.json` — registered `options_ui` pointing at `options.html`.
- `api/tailor.js` — added CORS headers (`Access-Control-Allow-Origin: *`, `OPTIONS` preflight
  handling) so a request from the extension's `chrome-extension://` origin isn't blocked. Safe here:
  no cookies/session are used, and the Anthropic API key never leaves the server.
- `extension/lib/settings.test.js` — 5 new unit tests (default-empty, trim/trailing-slash handling,
  resume text round-trip, undefined-safe).
- Docs updated: `CLAUDE.md`, `extension/README.md`, `autodev/PRODUCT_LOG.md`.

behind Settings: if the resume or web-app URL isn't set yet, `tailor.html` shows an inline notice
linking to Settings instead of silently failing.

## Follow-up: DOCX/PDF export (same day, requested directly by Muzaffar)

### What was built
- **Web app** (`src/ResumeTailor.jsx`): added "Download .docx" and "Download .pdf" buttons next to
  the existing Copy/.txt. Uses the `docx` and `jspdf` npm packages, both dynamically `import()`ed
  only when clicked (not in the initial bundle — `jspdf` alone pulls in `html2canvas`, ~500KB).
  PDF export paginates the resume text as real selectable Helvetica text (not a screenshot).
- **Extension** (`extension/tailor.js`): added "Download .docx" and "Print / Save as PDF" buttons.
  - `.docx` is hand-built: `extension/lib/docx.js` writes a minimal-but-valid OOXML `document.xml`
    (paragraphs, XML-escaped) and zips it via a **vendored local copy of JSZip**
    (`extension/lib/vendor/jszip.min.js`, 97KB minified) — MV3 CSP forbids pulling a library from a
    CDN at runtime, so it has to ship inside the extension.
  - `.pdf` uses the browser's native `window.print()` with a print-only stylesheet that hides
    everything except the resume text. No PDF library vendored — kept the extension light. Labelled
    "Print / Save as PDF" so the print-dialog behavior isn't a surprise.
- `extension/lib/docx.test.js` — 5 new tests that round-trip through the real `jszip` npm package
  (added as a devDependency) to verify the generated zip has the right parts, paragraph structure,
  blank-line handling, and XML escaping.
- `eslint.config.js` — excluded `extension/lib/vendor/` from linting (it's vendored/minified, not
  our code — linting it produced 136 unrelated errors).
- Docs updated again: `CLAUDE.md` (new "Resume export" section), `extension/README.md`,
  `autodev/PRODUCT_LOG.md`.

### Verification
