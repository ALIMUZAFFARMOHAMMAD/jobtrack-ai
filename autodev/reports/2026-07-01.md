# Autodev report — 2026-07-01

**Mode:** Feature (major) — new product direction set by Muzaffar.
**Built by:** this session (Muzaffar-directed), Phases 1 + 2. Agents own Phases 3–4.

## TL;DR
Shipped the **browser extension** for JobTrack AI: it tracks job applications while you
apply (hybrid auto-detect + one-click log), shows a toolbar badge with today's count, and
includes a dashboard with a 14-day chart, a table, and CSV export. All local, no backend.

## What was built
**Phase 1 — Extension MVP (tracking)**
- `extension/manifest.json` — MV3; content scripts on LinkedIn, Indeed, Greenhouse, Lever, Workday, Ashby.
- `extension/content/detect.js` — auto-detects "submit application / apply now / easy apply" clicks (plain "Submit" only on dedicated ATS domains); messages the worker.
- `extension/background.js` — service worker: stores events, dedupes, keeps the toolbar badge = today's count.
- `extension/lib/db.js` — storage/stats/CSV layer (dedupe = one per URL per day; streak; 14-day series).
- `extension/popup.*` — quick stats + one-click "Log application" (auto-fills role/company/URL from the tab).

**Phase 2 — Dashboard + export**
- `extension/dashboard.*` — today / last-7 / streak / total cards, 14-day bar chart, applications table with delete, and **CSV download** (opens in Excel/Sheets).

**Supporting**
- `eslint.config.js` — extension globals block (chrome/webextensions/serviceworker) so lint stays green.
- `extension/README.md` — install + how-it-works.

## Sentinel gate results
- `npm run build` (web app): ✅ pass (extension is separate, unaffected).
- `npm run lint`: ✅ pass (0 errors, incl. extension files).
- **Extension storage layer unit-tested in Node** with a mocked `chrome.storage`: add, dedupe-blocking, stats, streak, 14-day window, and CSV escaping all correct.
- Not verified in a live browser this session (no browser automation available); install steps in `extension/README.md`.

## Honest limitations
- Tracking is **in-browser only** — can't see phone/desktop-app applications.
- Auto-detect is best-effort per site; the one-click log is the reliable fallback; false positives are removable from the dashboard.
- Export is CSV (opens in Excel/Sheets); true `.xlsx` is a P4 item (vendor SheetJS).

# P3 — Resume tailoring (also shipped 2026-07-01)

**TL;DR:** Added AI resume tailoring to the **web app**. Paste a job description → Claude
returns an ATS-optimized rewrite of your saved resume + a match score + missing keywords +
suggestions, with copy/download.

## What was built
- `api/tailor.js` — serverless function; accepts `{resume, jobDescription}`, builds the ATS prompt
  **server-side**, returns `{atsResume, matchScore, missingKeywords[], suggestions[]}`. Robust JSON
  parsing with a raw-text fallback. API key never leaves the server. Model: `claude-sonnet-4-5`
  (matches the app's existing analyze call).
- `src/ResumeTailor.jsx` — self-contained "✨ Tailor Resume" view: JD textarea, match-score badge,
  keyword chips, suggestions, and the rewritten resume with Copy / Download .txt.
- `src/App.jsx` — 3 minimal edits: import, new tab button, render branch. (Kept isolated to limit
  blank-screen risk; guarded by the ErrorBoundary added 2026-06-30.)

## Sentinel gate results
- `npm run build`: ✅ (bundle 230→235 KB — ResumeTailor included).
- `npm run lint`: ✅ 0 errors.
- **Not** live-verified: the app is auth-gated and the preview tooling is anchored to the parent
  portfolio dir; and Vercel serverless functions + a real key don't run under local `vite dev`, so
  `/api/tailor` wasn't exercised end-to-end. Build compiles both files; endpoint logic reviewed.
